CIO-II

Writing assistance powered by Apple's on-chip AI — not the cloud. Watches you type, suggests corrections only when confident, and does nothing when uncertain.

The first open-source macOS application to use Apple's on-device Foundation Model (python-apple-fm-sdk / SystemLanguageModel) for real-time writing assistance.

Why You Would Want This

No More Silent Rewrites

CIO-II only suggests. You press Tab to accept or Esc to dismiss. It never changes text without showing you first.

Works Without Internet

Everything runs on your Mac's neural engine. No API keys, no subscriptions, no data leaving your machine.

Knows When to Shut Up

In code editors, terminals, password fields, and unknown apps — CIO-II does nothing. Backs off when you dismiss repeatedly.

Every Change Is Reversible

Every accepted suggestion has a tracked undo record. Press ctrl+option+z to reverse instantly.

AI That Cannot Invent

The Apple FM model can only pick from candidates CIO-II already identified — or return "do nothing." It never generates text.

Auditable by Design

Proof reports, privacy ledger, explain-last diagnostics. You can always see exactly what CIO-II did and why.

Real Examples

Typo in Apple Mail
You type: "I'll send teh report by end of day"
CIO-II suggests: the → Press Tab to accept, Esc to dismiss
Ambiguous correction (Apple FM decides)
You type "wierd" — two candidates: "weird" and "wired"
Apple FM on neural engine picks "weird" from context in <80ms. Or returns do_nothing if uncertain.
Code editor (VS Code, Xcode, PyCharm)
You write: def teh_function(): return wierd_value
CIO-II does nothing. Code profile = hard do_nothing. Your identifiers are never touched.
Password field
You click a login/password field in any browser
Hard block. No capture, no suggestions. Menu bar shows CIO-P (protected).
Dot-phrase expansion
You type .meW in an email
Expands to your full work signature. Secret fields (email, phone) resolve at apply-time from env vars.
Trust circuit breaker
You dismiss 3 suggestions in a row
CIO-II stops suggesting. Menu bar shows cooldown countdown. Resumes later with recalibrated thresholds.

Without the Apple FM SDK, CIO-II works in deterministic-only mode — it blocks ambiguous cases instead of routing to the arbiter.

Install

git clone https://github.com/deesatzed/CIO-II.git
cd CIO-II
./bootstrap.sh

bootstrap.sh creates the venv, installs dependencies, clones and builds the Apple FM SDK, and verifies your platform.

Then try it

# See all safety behaviors in a demo
./run_demo.sh

# Start native macOS mode (menu bar, hotkeys, real-time suggestions)
cio-ii run --mode mac

macOS will ask you to grant Accessibility permission. Allow your terminal in System Settings > Privacy & Security > Accessibility.

How the Apple FM Arbiter Works

You type "wierd" in an email
         |
    Candidate search finds: "weird" (0.82) and "wired" (0.79)
         |
    Confidence gap too small for deterministic choice
         |
    Route to Apple FM on neural engine:
    "Given context, which is correct? weird | wired | do_nothing"
         |
    FM responds within 80ms timeout
         |
    [weird] --> show suggestion     [do_nothing] --> show nothing
         |
    You press Tab or Esc

The FM model runs on your Mac's neural engine. It cannot invent text — it can only choose from the candidate list or return "do nothing." If it exceeds the 80ms timeout, the answer is always do_nothing. This is the fail-closed safety guarantee.
